Objetivo de Aprendizagem

FE1: Estruturar arquivos HTML a partir de sketches e restrições semânticas

Rubrica

Não Entregue (I) Em Desenvolvimento (D) Essencial (C) Proficiente (B) Avançado (A)
Nenhuma entrega. O conjunto de páginas não contempla integralmente os 3 casos de uso, apresenta estrutura confusa ou insuficiente. Uso excessivo de <div> sem função, sem hierarquia clara. Navegação interna ausente ou pouco coerente. Todas as páginas necessárias estão presentes, cobrindo integralmente os 3 casos de uso. Utiliza tags semânticas básicas (<header>, <nav>, <section>, <article>, <footer>) de forma coerente. Navegação interna funcional e elementos de interação adequados. Além do nível C, código segue padrões de qualidade (indentação, atributos adequados, validação W3C). Todas as páginas auxiliares também são implementadas. Além do nível B, páginas estruturadas de forma escalável e modular (design atômico), reutilização de componentes, uso de atributos de acessibilidade (ex.: aria-labelledby).

Exemplo de Uso

Casos de Uso

  1. Caso 1: Ana, aluna do Insper, precisa reservar uma sala para reunião da equipe. Acessa o calendário, seleciona dia 14 de maio, 14:00 - 16:00.
  2. Caso 2: Ana vê salas disponíveis, escolhe Sala 203, confirma reserva. Em seguida, edita reserva para a Sala 301 (maior capacidade) e confirma a alteração.
  3. Caso 3: Ana reporta uma ocorrência (TV da Sala 301 quebrada) via área de suporte, descreve o problema, anexa foto e envia relatório para manutenção.

Exemplos de Avaliação

Nível I (Não Entregue)

Sem código, pois não houve entrega.

Nível D (Em Desenvolvimento)

Características:

  • Cobertura incompleta dos casos de uso.
  • Semântica deficiente, uso excessivo de <div> genéricos.
  • Ausência de navegação interna coerente.
  • Fluxos não claramente conectados.
<!DOCTYPE html>
<html lang="pt-BR">
<head>
<meta charset="UTF-8">
<title>Aplicativo de Reservas</title>
</head>
<body>
    <div>
        <h1>Aplicativo de Reservas</h1>
        <a href="calendario.html">Calendário</a>
        <a href="reservar.html">Reservar</a>
        <a href="suporte.html">Suporte</a>
    </div>
</body>
</html>

Nível C (Essencial)

Características:

  • Todas as páginas necessárias estão presentes.
  • Utilização coerente de tags semânticas básicas.
  • Navegação interna funcional e elementos de interação adequados.
<!DOCTYPE html>
<html lang="pt-BR">
<head>
<meta charset="UTF-8">
<title>Aplicativo de Reservas - Nível C</title>
</head>
<body>
    <header>
        <h1>Aplicativo de Reservas</h1>
        <nav>
            <ul>
                <li><a href="calendario.html">Calendário</a></li>
                <li><a href="suporte.html">Suporte</a></li>
            </ul>
        </nav>
    </header>
    <main>
        <section>
            <h2>Casos de Uso</h2>
            <p>Selecionar a opção desejada no menu para iniciar o fluxo.</p>
        </section>
    </main>
    <footer>
        <p>© 2024 Insper</p>
    </footer>
</body>
</html>

Nível B (Proficiente)

Características:

  • Código segue padrões de qualidade (indentação, atributos adequados, validação W3C).
  • Todas as páginas auxiliares implementadas.
<!DOCTYPE html>
<html lang="pt-BR">
<head>
<meta charset="UTF-8">
<title>Aplicativo de Reservas - Nível B</title>
</head>
<body>
    <header>
        <h1>Aplicativo de Reservas</h1>
        <nav>
            <ul>
                <li><a href="calendario.html">Calendário</a></li>
                <li><a href="suporte.html">Suporte</a></li>
            </ul>
        </nav>
    </header>
    <main>
        <section>
            <h2>Salas Disponíveis</h2>
            <form>
                <label><input type="radio" name="sala" required> Sala 203</label><br>
                <label><input type="radio" name="sala" required> Sala 301</label><br>
                <button>Confirmar Reserva</button>
            </form>
        </section>
    </main>
    <footer>
        <p>© 2024 Insper</p>
    </footer>
</body>
</html>

Nível A (Avançado)

Características:

  • Organização do código em componentes reutilizáveis (design atômico).
  • Uso de atributos de acessibilidade, como aria-labelledby.
  • Estrutura escalável e modular, fácil de manter.
<!DOCTYPE html>
<html lang="pt-BR">
<head>
<meta charset="UTF-8">
<title>Aplicativo de Reservas - Nível A</title>
</head>
<body>
    <header>
        <h1 id="pageTitle">Aplicativo de Reservas</h1>
        <nav aria-labelledby="pageTitle">
            <ul>
                <li><a href="calendario.html">Calendário</a></li>
                <li><a href="suporte.html">Suporte</a></li>
                <li><a href="componentes.html">Componentes</a></li>
            </ul>
        </nav>
    </header>
    <main>
        <!-- Página organizada em "átomos", "moléculas" e "organismos" -->
        <section aria-labelledby="salasTitle">
            <h2 id="salasTitle">Salas Disponíveis</h2>
            <div class="lista-salas">
                <label>
                    <input type="radio" name="sala" value="203" required> Sala 203
                </label><br>
                <label>
                    <input type="radio" name="sala" value="301" required> Sala 301
                </label><br>
            </div>
            <div class="acao-reserva">
                <button>Confirmar Reserva</button>
            </div>
        </section>
        <section aria-labelledby="suporteTitle">
            <h2 id="suporteTitle">Suporte</h2>
            <form action="enviar-relato.html" method="post">
                <label for="descProblema">Descreva o problema:</label><br>
                <textarea id="descProblema" name="descricao" rows="4" cols="50" required></textarea><br>
                <label for="fotoProblema">Anexar foto:</label>
                <input type="file" id="fotoProblema" name="foto" accept="image/*"><br>
                <button type="submit">Enviar Relato</button>
            </form>
        </section>
    </main>
    <footer>
        <p>© 2024 Insper</p>
    </footer>
</body>
</html>

Checklist de Progresso

Use este checklist para orientar sua evolução de acordo com a rubrica, do nível D até A.

Conceito D (Em Desenvolvimento)

  • Criar os arquivos HTML iniciais, mesmo que simples.
  • Apresentar ao menos um fluxo básico do caso de uso principal.
  • Incluir links entre as páginas, ainda que de forma limitada.

Conceito C (Essencial)

  • Garantir que todas as páginas necessárias para os 3 casos de uso estejam presentes.
  • Utilizar tags semânticas básicas (ex: <header>, <nav>, <section>, <footer>).
  • Assegurar que os links e a navegação interna funcionem para percorrer os fluxos propostos.

Conceito B (Proficiente)

  • Aplicar boas práticas de código (indentação, atributos adequados) e passar em validações HTML.
  • Incluir atributos essenciais (ex: alt em imagens, required em campos de formulário) para qualidade e acessibilidade mínima.
  • Cobrir também páginas auxiliares, além das principais dos casos de uso.

Conceito A (Avançado)

  • Reestruturar o código com uma abordagem modular (design atômico: átomos, moléculas, organismos).
  • Aplicar atributos de acessibilidade (ex: aria-labelledby) para aprimorar a usabilidade por leitores de tela.
  • Garantir que componentes sejam reutilizáveis, escaláveis e mantenham padrão de qualidade uniforme.
