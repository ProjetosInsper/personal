FE1: Estruturar arquivos HTML a partir de sketches e restrições semânticas

Rubrica

Não Entregue (I) Em Desenvolvimento (D) Essencial (C) Proficiente (B) Avançado (A)
Nenhuma entrega. O conjunto de páginas não contempla integralmente os 3 casos de uso, apresenta estrutura confusa ou insuficiente. Uso excessivo de <div> sem função, sem hierarquia clara. Navegação interna ausente ou pouco coerente. Todas as páginas necessárias estão presentes, cobrindo integralmente os 3 casos de uso. Utiliza tags semânticas básicas (<header>, <nav>, <section>, <article>, <footer>) de forma coerente. Navegação interna funcional e elementos de interação adequados. Além do nível C, código segue padrões de qualidade (indentação, atributos adequados, validação W3C). Todas as páginas auxiliares também são implementadas. Além do nível B, páginas estruturadas de forma escalável e modular (design atômico), reutilização de componentes, uso de atributos de acessibilidade (ex.: aria-labelledby).

Exemplo de Uso

Casos de Uso

  1. Caso 1: Ana, aluna do Insper, precisa reservar uma sala para reunião da equipe. Acessa o calendário, seleciona dia 14 de maio, 14:00 - 16:00.
  2. Caso 2: Ana vê salas disponíveis, escolhe Sala 203, confirma reserva. Em seguida, edita reserva para a Sala 301 (maior capacidade) e confirma a alteração.
  3. Caso 3: Ana reporta uma ocorrência (TV da Sala 301 quebrada) via área de suporte, descreve o problema, anexa foto e envia relatório para manutenção.

Checklist para Avaliação

Conceito D (Em Desenvolvimento)

  • Implementar arquivos HTML básicos para o caso de uso 1.
  • Incluir elementos principais, ainda sem semântica ideal.
  • Links mínimos entre páginas, sem navegação detalhada.

Conceito C (Essencial)

  • Criar todas as páginas necessárias para os 3 casos de uso.
  • Utilizar tags semânticas básicas (<header>, <nav>, <footer>).
  • Garantir fluxos de navegação funcionais entre as páginas.

Conceito B (Proficiente)

  • Refinar o código seguindo padrões de qualidade (indentação, atributos adequados).
  • Adicionar atributos como alt em imagens e required em campos obrigatórios.
  • Validar o código HTML com as ferramentas W3C.

Conceito A (Avançado)

  • Organizar o código de forma escalável e modular (design atômico).
  • Utilizar atributos de acessibilidade como aria-labelledby.
  • Implementar práticas avançadas de design atômico.
